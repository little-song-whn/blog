VLA 模型在过去两年进展飞快。π₀、π₀.₅、OpenVLA…… 在一两步的桌面操作上,它们的成功率已经高得让人有点不敢相信。但只要任务一长,**复合误差就开始吃掉所有的胜率**。

我们这篇 Anticipation-VLA,本质上是在回答一个朴素的问题:**长程任务里,VLA 该怎么"看远一点"?**

## 一个具体的失败模式

想象一个任务:"把锤子从工具箱里拿出来,把钉子敲进木板"。

你给 VLA 这一句指令,它一上来就开始执行。前几步看起来都对 —— 走到工具箱,识别锤子,准备抓 —— 然后某一步抓偏了一点,锤柄歪了。**这一点点偏差,在接下来的"举起 → 移动 → 对准钉子 → 落锤"几十步里被累积放大**,最终落锤的位置离钉子十厘米远。

这是 VLA 的经典痛点。一个常见的解法是 **子任务分解**:把"敲钉子"拆成"抓锤子→举起→对准→敲下",每个子任务给底层策略一个明确的中间目标。

但这里藏着一个我们反复纠结的问题:**子任务该多细?**

## 固定粒度的两难

太粗,比如就分成"抓锤子→敲钉子"两段 —— 跟没分一样。

太细,比如分成"伸手 1cm→伸手 2cm→……→闭合夹爪 5%→……" —— 训练数据根本撑不起这种粒度,而且每个子目标的奖励信号微弱到没法学。

问题的本质是:**最优的子任务粒度依赖于当前状态**。距离远的时候应该粗,接近关键操作的时候应该细。固定粒度的方法注定是个折中。

所以我们想要的,是**能看着当前状态自适应地决定"下一个 milestone 该设在哪"** 的东西。

## 一个老朋友:贝尔曼分解

理论上其实有非常干净的支撑。考虑目标条件值函数,如果 $g'$ 在从 $s_0$ 到 $g$ 的最优路径上,那么:

$$
V^*(s_0, g) = V^*(s_0, g') + V^*(s_{g'}, g)
$$

也就是说,**任何"在最优路径上的中间状态"都能把全局最优值完美分解成两段之和**。

这个等式对我们意味着什么?它给了我们一个判断"中间子目标好不好"的客观标准 —— 一个好的中间子目标应该让两段值的和恰好等于全局值。

我们的预期模型(Anticipation Model)做的事情,就是去**学习生成这样的中间子目标**。

## 一个模型,四个角色

预期模型这件事比较有意思的地方在于,它需要同时具备四种能力:

1. **策略**:给定当前状态和最终目标,输出下一个文本子目标(比如"举起锤子到肩膀高度")。
2. **正向动力学**:对应这个文本子目标,生成视觉子目标图像 —— 让底层策略不仅听懂,还看到。
3. **逆向动力学**:从生成的图像反推动作或子目标,作为**自我判别正则化**,防止生成无意义的图像。
4. **价值评估**:判断当前状态距离子目标"还差多少",决定是否该换下一个子目标。

如果用四个独立模型,工程复杂度会爆炸,而且四者之间没有共享。我们最后选择**统一多模态模型(Bagel-based UMM)** 把四件事捏在一起 —— 一个模型,四个 head,联合训练。

```python
# 伪代码: 预期模型的四个角色共享底座
z = umm_backbone(obs, history, lang_goal)
text_subgoal  = head_policy(z)
image_subgoal = head_forward(z, text_subgoal)
verify        = head_inverse(image_subgoal, obs)   # 自我校验
value         = head_value(z)                      # 三分类: 达成/停滞/进展
```

价值这一项我们做了一个小但关键的改造:**把回归改成三分类**(目标达成 / 进展停滞 / 进展充分)。回归的训练信号在稀疏奖励下太弱,而三分类带来的判别信号干净得多 —— 这件事是踩了好几次坑之后才确定下来的。

## 子目标栈:能"反悔"的规划

光有"能生成子目标"还不够。真正的长程任务里,机器人会遇到**意外**:抓滑了、被挡住了、误判了距离。如果子目标一旦设定就不能改,机器人就会一直在死胡同里撞墙。

我们维护一个**子目标栈**,每 $K$ 步用价值模型判一次,触发三种操作:

- **Pop**:价值表示已达成,弹出当前子目标,回到上层目标。
- **Push**:进展停滞,生成更细粒度的子目标压栈。**栈满则回溯**,从初始状态重新规划。
- **Keep**:进展充分,继续。

这套机制很像我自己写代码时的状态:小步推进、卡住了拆得更细、彻底跑偏了就 git reset 重来。其实就是把人类规划时的递归 + 回溯写进了系统里。

## 实验里最让我意外的一点

LIBERO-Long 上的提升其实在我们意料之中,毕竟方法就是冲着长程去的。**真正让我惊讶的是 VLABench 的 Hammer Nail / Hang Picture 任务** —— 这类任务测试时用的是训练集外的图片,几乎所有现有方法在上面都是 **0%** 成功率。

Anticipation-VLA 是少数能把它从 0 拉起来的。这件事说明:**长程任务里"看不见的画面"才是真正的敌人**,而预期模型那种"先在脑子里画一张目标图"的能力,正好对应了人类应对新环境的策略 —— 先想清楚要什么样,再开始动手。

## 几个还没想清楚的问题

- **子目标栈的深度**:理论上可以无限递归,但实际上栈深超过 3 之后,效率就开始崩。我猜这和模型对"细到什么程度算一个原子动作"的理解有关。
- **价值模型的三分类边界**:目前是固定阈值,但不同任务的"停滞"判定其实差很多。是否应该让阈值本身变成可学习的?
- **视觉子目标的生成保真度**:Bagel 生图已经很好,但偶尔会幻觉出场景里不存在的物体。这种幻觉对底层策略是噪声还是误导,我们还没分得很清。
